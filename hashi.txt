# Deploying HashiCorp Vault Community Edition on Ubuntu 24.04 with Active Directory LDAP Authentication

This walkthrough will build a complete working lab deployment of:

* [HashiCorp Vault Community Edition](https://developer.hashicorp.com/vault?utm_source=chatgpt.com)
* Integrated Raft storage
* Systemd service
* Web UI
* Active Directory LDAP authentication
* AD group-to-policy mapping

This design fits very well into your existing:

* ADCS
* Active Directory
* PKI
* Machine identity
* Home lab environments

---

# Target Environment

Example environment used below:

| Component            | Example               |
| -------------------- | --------------------- |
| Ubuntu Server        | vault1.clientbank.com |
| Vault IP             | 10.0.25.240           |
| AD Domain            | clientbank.com        |
| Domain Controller    | dc1.clientbank.com    |
| LDAP Service Account | svc_vault_ldap        |
| Vault Admin AD Group | Vault-Admins          |

Adjust values for your environment.

---

# 1. Fresh Ubuntu Preparation

Open Terminal.

Update the OS:

```bash id="6m7ew6"
sudo apt update && sudo apt upgrade -y
```

Install required packages:

```bash id="twg7v0"
sudo apt install -y \
curl \
wget \
gpg \
unzip \
net-tools \
ca-certificates \
ldap-utils
```

---

# 2. Set Hostname

Set the server hostname:

```bash id="d3ul6h"
sudo hostnamectl set-hostname vault1.clientbank.com
```

Update `/etc/hosts`:

```bash id="yw8wp9"
sudo nano /etc/hosts
```

Example:

```text id="7g1u6o"
127.0.0.1 localhost
10.0.25.240 vault1.clientbank.com vault1
```

Verify:

```bash id="tr3yzn"
hostname -f
```

---

# 3. Configure DNS

Vault must resolve:

* Domain Controllers
* AD DNS names

Edit Netplan:

```bash id="ij7p6h"
sudo nano /etc/netplan/50-cloud-init.yaml
```

Example:

```yaml id="84wzc6"
network:
  version: 2
  ethernets:
    ens33:
      dhcp4: false
      addresses:
        - 10.0.25.240/24
      gateway4: 10.0.25.1
      nameservers:
        addresses:
          - 10.0.25.10
```

Apply:

```bash id="n6nchd"
sudo netplan apply
```

Test DNS:

```bash id="r82j1u"
nslookup dc1.clientbank.com
```

---

# 4. Join Ubuntu to Active Directory (Optional but Recommended)

This is optional for Vault LDAP auth itself, but useful operationally.

Install packages:

```bash id="zrptt7"
sudo apt install -y \
realmd \
sssd \
sssd-tools \
libnss-sss \
libpam-sss \
adcli \
samba-common-bin \
oddjob \
oddjob-mkhomedir \
packagekit
```

Discover the domain:

```bash id="zx3mbk"
realm discover clientbank.com
```

Join the domain:

```bash id="3h5s9e"
sudo realm join clientbank.com -U Administrator
```

Verify:

```bash id="r2ebzj"
realm list
```

Test AD login:

```bash id="3gw44k"
id administrator@clientbank.com
```

---

# 5. Add HashiCorp Repository

Import GPG key:

```bash id="rzv9r2"
wget -O- https://apt.releases.hashicorp.com/gpg | \
gpg --dearmor | \
sudo tee /usr/share/keyrings/hashicorp-archive-keyring.gpg > /dev/null
```

Add repository:

```bash id="79jbg7"
echo "deb [signed-by=/usr/share/keyrings/hashicorp-archive-keyring.gpg] \
https://apt.releases.hashicorp.com $(lsb_release -cs) main" | \
sudo tee /etc/apt/sources.list.d/hashicorp.list
```

Update:

```bash id="ol8wrv"
sudo apt update
```

---

# 6. Install Vault

Install:

```bash id="7lm0c6"
sudo apt install vault -y
```

Verify:

```bash id="pajyx8"
vault version
```

---

# 7. Create Vault Directories

```bash id="nvf7i5"
sudo mkdir -p /opt/vault/data
sudo mkdir -p /etc/vault.d
sudo mkdir -p /var/log/vault
```

Permissions:

```bash id="rlg57f"
sudo chown -R vault:vault /opt/vault
sudo chown -R vault:vault /var/log/vault
sudo chmod 750 /opt/vault/data
```

---

# 8. Create Vault Configuration

Create:

```bash id="u1g41d"
sudo nano /etc/vault.d/vault.hcl
```

Paste:

```hcl id="5pb4cq"
ui = true

disable_mlock = true

storage "raft" {
  path = "/opt/vault/data"
  node_id = "vault1"
}

listener "tcp" {
  address     = "0.0.0.0:8200"
  tls_disable = 1
}

api_addr = "http://vault1.clientbank.com:8200"
cluster_addr = "http://vault1.clientbank.com:8201"
```

Save.

Set permissions:

```bash id="gvsav9"
sudo chown vault:vault /etc/vault.d/vault.hcl
sudo chmod 640 /etc/vault.d/vault.hcl
```

# 16. Create Active Directory Service Account

In Active Directory create:

| Object                 | Example        |
| ---------------------- | -------------- |
| User                   | svc_vault_ldap |
| Password Never Expires | Yes            |
| Interactive Login      | Disabled       |

Recommended permissions:

* Read-only directory access
* No admin rights

---

# 17. Create Vault AD Security Group

Create AD group:

```text id="7qez4y"
Vault-Admins
```

Add your admin user into the group.

---

# 18. Enable LDAP Authentication

Enable LDAP auth in Vault:

```bash id="plx4ec"
vault auth enable ldap
```

---

# 19. Configure Vault LDAP Authentication

Run:

```bash id="14jvct"
vault write auth/ldap/config \
url="ldaps://dc1.clientbank.com" \
binddn="CN=svc_vault_ldap,CN=Users,DC=clientbank,DC=com" \
bindpass='YourPasswordHere' \
userdn="CN=Users,DC=clientbank,DC=com" \
groupdn="CN=Users,DC=clientbank,DC=com" \
userattr="sAMAccountName" \
groupattr="cn" \
certificate=@/etc/ssl/certs/clientbank-root-ca.pem \
insecure_tls=false \
starttls=false
```

---

# 20. Import Your ADCS Root CA Certificate

Export your Root CA certificate from ADCS.

Copy to Ubuntu:

```bash id="cgv8iz"
sudo cp clientbank-root-ca.pem /usr/local/share/ca-certificates/
```

Update trust:

```bash id="ag6m0h"
sudo update-ca-certificates
```

Test LDAPS:

```bash id="y93go7"
openssl s_client -connect dc1.clientbank.com:636
```

You should see:

* successful TLS negotiation
* certificate chain

---

# 21. Create Vault Admin Policy

Create policy file:

```bash id="8mjlwm"
nano admin-policy.hcl
```

Paste:

```hcl id="6zjlwm"
path "*" {
  capabilities = ["create", "read", "update", "delete", "list", "sudo"]
}
```

Load policy:

```bash id="gnhymz"
vault policy write admin admin-policy.hcl
```

---

# 22. Map AD Group to Vault Policy

Map AD group:

```bash id="yfrvqk"
vault write auth/ldap/groups/Vault-Admins policies=admin
```

---

# 23. Test Active Directory Authentication

Logout root token:

```bash id="7c8lx8"
vault token revoke -self
```

Login using AD:

```bash id="u5q9s4"
vault login -method=ldap username=yourusername
```

Vault prompts for:

* AD password

If successful:

* You receive Vault token
* Policies assigned from AD group

---

# 24. Enable KV Secrets Engine

Enable secrets:

```bash id="dg8n44"
vault secrets enable -path=secret kv-v2
```

Store secret:

```bash id="f9n77o"
vault kv put secret/test username=test password=Password123
```

Retrieve:

```bash id="ibf4lw"
vault kv get secret/test
```
