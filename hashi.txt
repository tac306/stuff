Yes. Below is a clean installation process for deploying [HashiCorp Vault Community Edition](https://developer.hashicorp.com/vault?utm_source=chatgpt.com) on a fresh Ubuntu system.

This walkthrough will:

* Install Vault from HashiCorp’s official repository
* Configure Vault as a systemd service
* Use integrated Raft storage
* Enable TLS-ready configuration structure
* Initialize and unseal Vault
* Verify web UI and CLI access

This is appropriate for:

* Home lab
* Development
* PKI testing
* Learning environments
* Small internal deployments

For production, you would later add:

* Real TLS certificates
* Auto-unseal (HSM/KMS)
* Backup strategy
* HA clustering
* Audit logging

---

# 1. Update Ubuntu

Open a terminal:

```bash
sudo apt update && sudo apt upgrade -y
```

Install utilities:

```bash
sudo apt install -y gpg curl wget unzip net-tools
```

---

# 2. Add the HashiCorp Repository

Import the GPG key:

```bash
wget -O- https://apt.releases.hashicorp.com/gpg | \
gpg --dearmor | \
sudo tee /usr/share/keyrings/hashicorp-archive-keyring.gpg > /dev/null
```

Add the repository:

```bash
echo "deb [signed-by=/usr/share/keyrings/hashicorp-archive-keyring.gpg] \
https://apt.releases.hashicorp.com $(lsb_release -cs) main" | \
sudo tee /etc/apt/sources.list.d/hashicorp.list
```

Update package lists:

```bash
sudo apt update
```

---

# 3. Install Vault

Install the Community Edition package:

```bash
sudo apt install vault -y
```

Verify:

```bash
vault version
```

You should see something similar to:

```text
Vault v1.x.x
```

---

# 4. Create Vault Directories

Create required directories:

```bash
sudo mkdir -p /opt/vault/data
sudo mkdir -p /etc/vault.d
sudo mkdir -p /var/log/vault
```

Set permissions:

```bash
sudo chown -R vault:vault /opt/vault
sudo chown -R vault:vault /var/log/vault
sudo chmod 750 /opt/vault/data
```

---

# 5. Create the Vault Configuration File

Create the main configuration:

```bash
sudo nano /etc/vault.d/vault.hcl
```

Paste this configuration:

```hcl
ui = true

disable_mlock = true

storage "raft" {
  path = "/opt/vault/data"
  node_id = "vault-node1"
}

listener "tcp" {
  address     = "0.0.0.0:8200"
  tls_disable = 1
}

api_addr = "http://127.0.0.1:8200"
cluster_addr = "http://127.0.0.1:8201"
```

Save and exit.

---

# 6. Secure the Configuration File

```bash
sudo chown vault:vault /etc/vault.d/vault.hcl
sudo chmod 640 /etc/vault.d/vault.hcl
```

---

# 7. Enable and Start Vault

Enable the service:

```bash
sudo systemctl enable vault
```

Start Vault:

```bash
sudo systemctl start vault
```

Check status:

```bash
sudo systemctl status vault
```

You should see:

```text
active (running)
```

---

# 8. Configure the Vault CLI Environment Variable

Add the Vault address for your shell:

```bash
echo 'export VAULT_ADDR=http://127.0.0.1:8200' >> ~/.bashrc
source ~/.bashrc
```

Verify connectivity:

```bash
vault status
```

At this stage, Vault should report:

```text
Initialized false
Sealed true
```

---

# 9. Initialize Vault

Initialize the Vault instance:

```bash
vault operator init
```

You will receive:

* 5 unseal keys
* 1 initial root token

Example:

```text
Unseal Key 1: xxxxx
Unseal Key 2: xxxxx
...
Initial Root Token: hvs.xxxxx
```

IMPORTANT:

* Save these securely
* Losing them can permanently lock you out
* Do not leave them in terminal history or screenshots

For a lab:

* Save to an encrypted password manager
* Or an offline text file

---

# 10. Unseal Vault

Use 3 unseal keys:

```bash
vault operator unseal
```

Paste key #1.

Repeat:

```bash
vault operator unseal
```

Paste key #2.

Repeat again with key #3.

Check status:

```bash
vault status
```

Now you should see:

```text
Sealed false
```

---

# 11. Login to Vault

Authenticate using the root token:

```bash
vault login
```

Paste the root token.

---

# 12. Access the Web UI

Open a browser on the Ubuntu desktop:

```text
http://localhost:8200
```

Or remotely:

```text
http://<server-ip>:8200
```

Login using:

* Token authentication
* Root token

---

# 13. Enable a Secrets Engine (Example)

Enable KV secrets:

```bash
vault secrets enable -path=secret kv-v2
```

Store a secret:

```bash
vault kv put secret/test username=admin password=SuperSecret123
```

Retrieve it:

```bash
vault kv get secret/test
```

# 14. Recommended Next Steps

## Enable TLS

Right now Vault is running without encryption:

```hcl
tls_disable = 1
```

For anything beyond isolated lab usage:

* Create a certificate
* Enable HTTPS
* Use proper DNS names

You could later use:

* Self-signed certs
* Your ADCS lab
* Venafi-managed certificates
* Let’s Encrypt

---

## Enable Audit Logging

Create an audit log:

```bash
sudo mkdir -p /var/log/vault_audit
sudo chown vault:vault /var/log/vault_audit
```

Enable auditing:

```bash
vault audit enable file file_path=/var/log/vault_audit/audit.log
```

---

## Enable PKI Secrets Engine

Vault is extremely useful as an internal CA.

Enable PKI:

```bash
vault secrets enable pki
```

Tune TTL:

```bash
vault secrets tune -max-lease-ttl=87600h pki
```

This is where Vault becomes very useful for:

* Dynamic certificates
* Short-lived certificates
* Lab PKI
* Kubernetes identities
* Machine identity automation
